# Roadmap

This roadmap lists planned portfolio and engineering improvements. It does not claim that the listed items are already implemented.

## Documentation

- Add screenshots to README
- Add short animated or step-by-step usage examples
- Add anonymized sample Excel output
- Add configuration file example without sensitive data
- Improve setup instructions after testing on a clean machine

## Desktop App

- Split Python desktop application into smaller modules
- Add logging improvements
- Add user-friendly error messages
- Add clearer handling for portal layout changes
- Improve packaging notes for Windows releases

## Android App

- Improve Android UI polish
- Add clearer status messages for long-running extraction
- Document release signing flow without exposing private keys
- Test on multiple Android screen sizes

## Data Export

- Add basic automated tests for data mapping
- Add anonymized sample workbook for documentation
- Document column meanings in a separate data dictionary
- Add validation checks for key exported fields

## Code Quality

- Add basic automated tests for pure mapping helpers
- Add formatting and linting notes
- Introduce smaller modules around extraction, mapping and export
- Keep platform-specific behavior documented

## Future Improvements

- Publish APK and Windows build through GitHub Releases
- Add optional configuration file example without sensitive data
- Add a troubleshooting guide for browser, WebView and export issues
- Add a changelog for portfolio releases
